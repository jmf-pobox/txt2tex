\section*{Bags (Multisets)}
\addcontentsline{toc}{section}{Bags (Multisets)}

\subsection*{Example 1 : Bag Type}
\addcontentsline{toc}{subsection}{Example 1 : Bag Type}

\noindent Bags are unordered collections that allow duplicates:

\bigskip

\noindent
$\bag~\nat$


\noindent
$\bag~\num$


\subsection*{Example 2 : Bag Literals}
\addcontentsline{toc}{subsection}{Example 2 : Bag Literals}

\noindent Bags are written with double square brackets:

\bigskip

\noindent
$\lbag x \rbag$


\noindent
$\lbag a, b, c \rbag$


\noindent
$\lbag 1, 2, 2, 3, 3, 3 \rbag$


\subsection*{Example 3 : Bags vs Sets}
\addcontentsline{toc}{subsection}{Example 3 : Bags vs Sets}

\noindent Unlike sets, bags preserve multiplicity:

\bigskip

\noindent The bag [[1, 2, 2, 3]] is different from the set $\{1, 2, 3\}$

\bigskip

\noindent The bag [[a, a, a]] contains three copies of a

\bigskip

\subsection*{Example 4 : Bags elem Specifications}
\addcontentsline{toc}{subsection}{Example 4 : Bags elem Specifications}

\noindent Bags can model collections where order doesn't matter but quantity does:

\bigskip

\noindent
$coins \in \bag~Coin$


\noindent
$items \in \bag~Item$
