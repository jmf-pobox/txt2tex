// Foreign Keys (Phase 3.2)
// Four patterns: single-attr PK, composite PK, single-attr FK,
// composite FK + self-referential FK.
// Domain: kitchen-management (Recipe, Ingredient, RecipeIngredient, Employee)

TITLE: Foreign Keys in Relational Schemas
AUTHOR: txt2tex example

given RecipeId, IngredientId, EmployeeId, CuisineTag, IngredientName, RecipeName, EmployeeName, PositionName

LINEBREAK:

=== Pattern 1: Single-Attribute Primary Key ===

TEXT: Each relation has exactly one pk line.  txt2tex underlines the attribute
TEXT: in the schema box and emits a PK statement immediately after.

schema Recipe
  pk recipeId : RecipeId
  name : RecipeName
  cuisine : CuisineTag
end

schema Ingredient
  pk ingredientId : IngredientId
  name : IngredientName
end

LINEBREAK:

=== Pattern 2: Composite Primary Key ===

TEXT: Two pk lines in one schema body.  Both attributes are underlined;
TEXT: txt2tex collapses them into one PK set: PK(RecipeIngredient) = {recipeId, ingredientId}.

schema RecipeIngredient
  pk recipeId : RecipeId
  pk ingredientId : IngredientId
end

LINEBREAK:

=== Pattern 3: Single-Attribute Foreign Key ===

TEXT: A foreign key constraint says every value of the referencing attribute
TEXT: must appear in the referenced relation.  No special keyword is needed:
TEXT: write the constraint as a Z predicate using forall/exists or
TEXT: projection with subset.

TEXT: RecipeIngredient.recipeId must reference an existing Recipe:

forall ri : RecipeIngredient | exists r : Recipe | ri.recipeId = r.recipeId

TEXT: Equivalently, using projection and subset:

pi[recipeId](RecipeIngredient) subset pi[recipeId](Recipe)

TEXT: RecipeIngredient.ingredientId must reference an existing Ingredient:

forall ri : RecipeIngredient | exists i : Ingredient | ri.ingredientId = i.ingredientId

LINEBREAK:

=== Pattern 4: Composite FK and Self-Referential FK ===

TEXT: Both FK constraints for RecipeIngredient can be stated together
TEXT: in a single zed block.  Each predicate is separated by an also
TEXT: command in the output.

zed
  forall ri : RecipeIngredient | exists r : Recipe | ri.recipeId = r.recipeId

  forall ri : RecipeIngredient | exists i : Ingredient | ri.ingredientId = i.ingredientId
end

TEXT: Employee carries a managerId that references another row in the
TEXT: same relation.  The self-referential FK constraint:

schema Employee
  pk employeeId : EmployeeId
  name : EmployeeName
  position : PositionName
  managerId : EmployeeId
end

TEXT: Every managerId in Employee must match a valid employeeId in the
TEXT: same relation:

zed
  forall e : Employee | exists m : Employee | e.managerId = m.employeeId
end
