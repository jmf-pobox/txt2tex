// Features:
//   - Column-aligned layout with & separators
//   - Single-line and multi-line definitions
//   - Blank line grouping with \also
//   - Parameterized constructors with angle brackets

=== Syntax Environment Examples ===

zed
  given NAME
end

** Example 1: Simple Enumeration **

TEXT: For simple free types with several branches, syntax provides
TEXT: column-aligned layout:

syntax
  OP1 ::= plus | minus | times | divide
end

TEXT: This generates aligned columns with the ::= separator clearly visible.

** Example 2: Parameterized Constructors **

TEXT: Constructors can have type parameters using angle brackets:

syntax
  Tree ::= leaf<N> | branch<Tree cross Tree>
end

TEXT: Parameters are rendered with special data delimiters in LaTeX.

** Example 3: Multiple Definitions with Grouping **

TEXT: Blank lines between definitions create visual groups using \textbackslash also:

syntax
  Status2 ::= active2 | inactive2 | pending2

  Response ::= ok | error<N> | timeout
end

TEXT: The blank line causes the also command to be generated, creating vertical
TEXT: separation between the two type definitions.

** Example 4: Complex BNF-Style Definitions **

TEXT: Syntax environment is ideal for BNF-style grammar definitions:

syntax
  Expr ::= const<N> | var<NAME> | binop<OP1 cross Expr cross Expr>

  Stmt ::= assign<NAME cross Expr> | seq<Stmt cross Stmt> | skip
end

TEXT: This creates a professionally formatted grammar specification.

** Comparison: Syntax vs Zed Blocks **

TEXT: Standard zed blocks work for simple types:

zed
  Color1 ::= red1 | green1 | blue1
end

TEXT: But syntax environment provides better alignment for complex types:

syntax
  Color2 ::= red2 | green2 | blue2

  Shape ::= circle<N> | rectangle<N cross N> | polygon<seq N>
end

TEXT: The column alignment makes the structure more readable.
